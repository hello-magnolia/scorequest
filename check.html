Sprite transparency check, with file fingerprinting

Each frame is fetched fresh from THIS server, fingerprinted (SHA-256), and compared to the exact bytes committed in the repo, then drawn over magenta and tested for background residue. Three verdicts are possible per frame: PASS (current file, clean), OUTDATED (this server delivered a file that does not match the repo, so the fix exists but is not reaching you), and FAIL (the file matches the repo and still has residue, which means the repo itself is at fault).
